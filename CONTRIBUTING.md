## How to Contribute
1. **Fork the repository**  
2. **Create a new branch** for your work  
```git checkout -b my-feature```
3. **Add your content** inside `/templates` (if creating a new template) or `/content` (if submitting a draft)  
4. **Use the appropriate Issue Template**:
- `Content Piece` for blogs, newsletters, social posts
- Reference a strategy or portfolio issue if needed
5. **Open a Pull Request (PR)**  
- Link the issue you are working on  
- Add a descriptive title and summary

## Style Guidelines
- Keep Markdown clean and structured  
- Use headings consistently (#, ##, ###)  
- Use lists for clarity  
- Add links, images, and visuals when relevant  

## Review Process
- PRs will be reviewed by founders or core team  
- Feedback will be provided in the PR comments  
- Approved PRs will be merged into the main branch

## Code of Conduct
Be respectful, constructive, and collaborative.
